## Development Setup

1. Fork the repository
2. Clone your fork: `git clone git@github.com:Kxpi/yadr.git`
3. Install dependencies: `npm install`
4. Create a branch: `git checkout -b feature/your-feature-name`
5. Make your changes
6. Test: `npm run build`
7. Commit: `git commit -m "feat: your feature description"`
8. Push: `git push origin feature/your-feature-name`
9. Open a Pull Request

## Commit Convention

We use conventional commits:

- `feat:` - New feature
- `fix:` - Bug fix
- `docs:` - Documentation changes
- `style:` - Code style changes (formatting, etc.)
- `refactor:` - Code refactoring
- `test:` - Adding tests
- `chore:` - Maintenance tasks

Examples:
```
feat: add blog section
fix: mobile navigation not closing
docs: update deployment instructions
```

## Pull Request Guidelines

- Keep PRs focused on a single feature/fix
- Update documentation if needed
- Make sure `npm run build` works
- Test on mobile and desktop
- Describe what changed and why

## Questions?

Open a discussion or issue! We're here to help.

## Code of Conduct

Be respectful, inclusive, and constructive. This is a welcoming space for everyone.
